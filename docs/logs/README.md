# 開発ログ（会話履歴）

このディレクトリには、DTM Web Appの開発過程における重要な会話、設計決定、議論の記録を保存します。

## 目的

1. **コンテキストの継続性**: セッションが変わっても過去の議論を参照できる
2. **設計の意図の保存**: なぜその設計にしたのかを記録
3. **開発の連続性**: 前回どこまで進んだか、次に何をするかを明確に
4. **意思決定の記録**: 重要な技術選択や方針変更の理由を保存

## ファイル命名規則

```
YYYY-MM-DD-session-NN.md

例:
2025-11-06-session-01.md
2025-11-07-session-01.md
2025-11-07-session-02.md
```

## ログに含めるべき内容

### 必須項目
- 日付・時刻
- セッションの目標
- 実施した作業
- 作成したファイル・ドキュメント
- 重要な技術的決定
- 次回のアクション

### オプション項目
- 議論のサマリー
- 遭遇した問題と解決策
- パフォーマンスに関するメモ
- リファクタリングの候補
- 今後の課題

## 使い方

### 新しいセッション開始時

1. 最新のログファイルを開く
2. 前回の「次回のアクション」を確認
3. 今日の目標を設定

### セッション終了時

1. 今日の成果をログに記録
2. 次回のアクションを明記
3. Git commit

### 過去のログを参照したい時

1. ファイル名から日付で検索
2. または `grep -r "キーワード" docs/logs/` で全文検索

## テンプレート

各ログファイルは以下のテンプレートに従います：

```markdown
# セッション: YYYY-MM-DD-NN

## 📅 日時
- 日付: YYYY年MM月DD日
- 時間: HH:MM - HH:MM
- セッション: N回目

## 🎯 目標
今日達成したいこと

## 📝 実施内容
実際に行った作業

## 📄 作成・更新ファイル
- ファイル名 (説明)

## 💡 重要な決定事項
技術的な決定、方針変更など

## 🔧 遭遇した問題と解決策
問題とその解決方法

## ✅ 達成したこと
完了したタスク

## 📋 次回のアクション
次にやるべきこと
```

## ログの管理

### 定期的なレビュー
- 週次: その週の進捗を振り返る
- 月次: マイルストーンの達成度を確認

### アーカイブ
- 3ヶ月以上前のログは `docs/logs/archive/` に移動

### 検索性の向上
- 重要なキーワードにはタグを付ける: `#database #ui #performance`

---

**このログシステムにより、開発の連続性と品質を維持します。**
